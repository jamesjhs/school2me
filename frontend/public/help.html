School2Me Help Center

This standalone guide explains all user-facing features of School2Me. It is intentionally public and can be opened directly at /help.html without logging in.

What School2Me Does

School2Me centralizes school communications into a family timeline. You can sign in without a password, invite family members, track events per child/activity, and manage admin delivery/webhook settings.

Authentication Features

1) Magic Link Login

  1. Go to the Auth page.
  2. Select Magic Link.
  3. Enter your email and Turnstile response.
  4. Request link, then verify with the token from your email.

2) Email 2FA Login

  1. Select Email 2FA.
  2. Enter email and Turnstile response.
  3. Request code and submit the 6-digit code.

3) Join Family (Share Name + Password)

  1. Select Join (Share Name + Password).
  2. Enter your email, share name, and share password.
  3. Submit to join the family account.

4) Join Family (Share Link Token)

  1. Select Join (Share Link/QR Token).
  2. Enter your email and token from share link/QR text.
  3. Submit to join.

Dashboard (Family Timeline)

  • View upcoming parsed events grouped by day.
  • Filter by child or by activity.
  • See title, schedule, location, description, and “Needs Review” indicator.

Settings

Children

  • Add child records (name, class, teacher).
  • Review existing child list.

Activities

  • Add activities to a specific child.
  • Supported activity types: school club, hobby, sports, volunteering.

Invites

  • Generate share credentials for family onboarding.
  • Receive share link and token text for QR workflows.

Admin Panel

  • Separate admin login (email + password).
  • Configure SMTP host/port/user/pass.
  • Configure webhook API key used by inbound email webhook.

Public Integrations

  • Calendar feed: /feeds/<routing_alias>/calendar.ics
  • RSS feed: /feeds/<routing_alias>/rss
  • Join link handoff: /join/<token>

Troubleshooting

  • Unauthorized: session cookie may be missing/expired; log in again.
  • CSRF errors: reload app to refresh token cookie and retry.
  • Turnstile failed: verify token is valid and keys are configured.
  • No events: check webhook key and inbound payload fields.
  • No email delivery: verify SMTP settings in Admin.
